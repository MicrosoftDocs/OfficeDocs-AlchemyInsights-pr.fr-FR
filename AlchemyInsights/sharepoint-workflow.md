# <a name="workflows-in-sharepoint"></a>Flux de travail dans SharePoint

Les flux de travail SharePoint sont des applications préprogrammées qui rationalisent un grand nombre de processus d’entreprise. Pour plus d’informations, consultez les liens ci-dessous.

## <a name="getting-started"></a>Mise en route

Si vous commencez à utiliser les flux de travail SharePoint, voir [Introduction aux flux de travail SharePoint](https://support.office.com/article/introduction-to-sharepoint-workflow-07982276-54e8-4e17-8699-5056eff4d9e3).

## <a name="troubleshoot-and-debug-a-sharepoint-workflow"></a>Dépanner et déboguer un flux de travail SharePoint

Voir [résoudre les erreurs de validation de flux de travail SharePoint](https://docs.microsoft.com/sharepoint/dev/general-development/troubleshooting-sharepoint-server-workflow-validation-errors-in-visio) et [déboguer les flux de travail SharePoint](https://docs.microsoft.com/sharepoint/dev/general-development/debugging-sharepoint-server-workflows).

## <a name="power-automate"></a>Automate d’alimentation

Envisagez [d’utiliser Power automate](https://docs.microsoft.com/power-automate/modern-approvals) à la place des solutions de flux de travail SharePoint classiques.
