[<span data-ttu-id="48d1c-103">VIDÉO : Réparation d’un compte compromis</span><span class="sxs-lookup"><span data-stu-id="48d1c-103">VIDEO: Fixing a compromised account</span></span>](https://www.microsoft.com/videoplayer/embed/RE2jvOb?pid=ocpVideo0-innerdiv-oneplayer&amp;postJsllMsg=true&amp;maskLevel=20&amp;autoplay=true)
  
1. <span data-ttu-id="48d1c-104">[Réinitialisez immédiatement le mot de passe de l’utilisateur](https://docs.microsoft.com/microsoft-365/admin/add-users/reset-passwords).</span><span class="sxs-lookup"><span data-stu-id="48d1c-104">[Reset the user's password](https://docs.microsoft.com/microsoft-365/admin/add-users/reset-passwords) immediately.</span></span> <span data-ttu-id="48d1c-105">Ne transmettez pas le nouveau mot de passe par courrier électronique à l’utilisateur final.</span><span class="sxs-lookup"><span data-stu-id="48d1c-105">Do not communicate the new password through email to the end user.</span></span>

2. <span data-ttu-id="48d1c-106">Supprimez les [adresses de transfert](https://docs.microsoft.com/microsoft-365/admin/email/configure-email-forwarding) suspectes définies au niveau de la boîte aux lettres.</span><span class="sxs-lookup"><span data-stu-id="48d1c-106">Remove any suspicious [forwarding addresses](https://docs.microsoft.com/microsoft-365/admin/email/configure-email-forwarding) set at the mailbox level.</span></span>

3. <span data-ttu-id="48d1c-107">Supprimez les [règles de boîte de réception](https://support.office.com/article/1433E3A0-7FB0-4999-B536-50E05CB67FED) suspectes définies dans la boîte aux lettres.</span><span class="sxs-lookup"><span data-stu-id="48d1c-107">Remove any suspicious [inbox rules](https://support.office.com/article/1433E3A0-7FB0-4999-B536-50E05CB67FED) set within the mailbox.</span></span>

4. <span data-ttu-id="48d1c-108">Si l’utilisateur ne peut pas envoyer d’e-mail, [accédez à la page Utilisateurs restreints pour débloquer le compte](https://protection.office.com/?hash=/restrictedusers).</span><span class="sxs-lookup"><span data-stu-id="48d1c-108">If the user is blocked from sending email, [go to the Restricted Users page to unblock the account](https://protection.office.com/?hash=/restrictedusers).</span></span> <span data-ttu-id="48d1c-109">Une fois cette opération effectuée, l’utilisateur doit pouvoir reprendre l’envoi de messages dans l’heure qui suit.</span><span class="sxs-lookup"><span data-stu-id="48d1c-109">Once done, the user should be able to resume sending messages within 1 hour.</span></span>

5. <span data-ttu-id="48d1c-110">Supprimez le compte d’utilisateur de tout [groupe doté d’un rôle administratif](https://docs.microsoft.com/microsoft-365/admin/add-users/assign-admin-roles) jusqu’à ce que vous soyez certain qu’il n’est plus compromis.</span><span class="sxs-lookup"><span data-stu-id="48d1c-110">Remove the user account from any [administrative role groups](https://docs.microsoft.com/microsoft-365/admin/add-users/assign-admin-roles) until you are confident that the account is no longer compromised.</span></span>

<span data-ttu-id="48d1c-111">Pour minimiser le risque de violation de données ou de compromission du compte à l’avenir, nous vous recommandons de lire notre [feuille de route de sécurité Microsoft 365](https://docs.microsoft.com//office365/securitycompliance/security-roadmap).</span><span class="sxs-lookup"><span data-stu-id="48d1c-111">To minimize the potential of a data breach or a compromised account in the future, we recommend reading our [Microsoft 365 security roadmap ](https://docs.microsoft.com//office365/securitycompliance/security-roadmap).</span></span>
