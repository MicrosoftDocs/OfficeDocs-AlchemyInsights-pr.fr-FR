# <a name="encrypt-email-messages-in-outlook"></a>Chiffrer les messages électroniques dans Outlook

<p><span style="font-size: 10.5pt; font-family: 'Verdana',sans-serif;">Le chiffrement de messages Office 365 est basé sur Microsoft Azure Rights Management (Azure RMS) qui fait partie d’Azure information protection. Si votre abonnement inclut Azure Rights Management ou Azure information protection, <strong style="mso-bidi-font-weight: normal;">il n’est pas nécessaire d’effectuer des actions pour activer ou activer manuellement</strong> le service gestion des droits.</span></p> <p><span style="font-size: 10.5pt; font-family: 'Verdana',sans-serif;">En fonction des commentaires des clients, nous n’allons plus activer les règles de flux de messagerie Exchange pour chiffrer automatiquement les messages électroniques sortants contenant certains types d’informations sensibles dans votre client par défaut. &nbsp; Au lieu de cela, nous fournissons des instructions détaillées pour vous permettre de le faire. &nbsp;Pour plus d’informations sur la création d’une règle de transport pour chiffrer des informations sensibles, consultez cet <a href="https://aka.ms/OmeEtr">article</a>.</span><u></u><span style="text-decoration: line-through;"></span></p> <ul> <li style="text-indent: -.25in; mso-list: l0 level1 lfo1;"><span style="font-size: 10.5pt; font-family: Symbol; mso-fareast-font-family: Symbol; mso-bidi-font-family: Symbol;"><span style="mso-list: Ignore;">&nbsp;Si &nbsp; utilisez Outlook sur le Web (anciennement OWA): lors de la composition d’un message électronique, cliquez simplement sur protéger dans OWA. &nbsp; &nbsp; </span> </span> <span style="font-size: 10.5pt; font-family: 'Verdana',sans-serif;"> <strong style="mso-bidi-font-weight: normal;"></strong> <strong></strong> Cette opération s’applique par &lsquo;défaut ne pas&rsquo; transférer l’autorisation. Cliquez sur <strong>modifier l’autorisation</strong> et sélectionnez <strong>chiffrer</strong> pour uniquement chiffrer le message.</span></li> <li style="text-indent: -.25in; mso-list: l0 level1 lfo1;"><span style="font-size: 10.5pt; font-family: 'Verdana',sans-serif;">&nbsp;Si vous utilisez le <strong style="mso-bidi-font-weight: normal;">client Outlook</strong>: pour envoyer un message chiffré à partir d’Outlook 2013 ou 2016, ou Outlook 2016 pour Mac &agrave; , sélectionnez Options autorisations, puis sélectionnez l’option de protection dont vous avez besoin. &nbsp; &nbsp;</span></li> <li style="text-indent: -.25in; mso-list: l0 level1 lfo1;"><span style="font-size: 10.5pt; font-family: 'Verdana',sans-serif;">&nbsp;&nbsp; Pour <strong style="mso-bidi-font-weight: normal;">chiffrer automatiquement tous les messages électroniques</strong> envoyés à certains destinataires ou organisations partenaires externes, vous devez créer une règle de transport de flux de messagerie dans le centre d’administration Exchange. Des instructions détaillées sont fournies dans cet <span style="color: black;"><a href="https://docs.microsoft.com/office365/securitycompliance/define-mail-flow-rules-to-encrypt-email#create-a-mail-flow-rule-to-encrypt-email-messages-with-the-new-ome-capabilities">article de support technique</a></span> .</span></li> </ul>
